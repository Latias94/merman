- Done: `workspace/symbol` now reuses tracked outline projections from document snapshots; semantic tokens have full/range/delta handlers over parser-backed snapshot tokens; configuration changes trigger `workspace/semanticTokens/refresh` when supported; U3 navigation and rename route through entity-only typed reference groups; directive-oriented completion contexts no longer fall back to generic `flowchart TD` on directive lines; flowchart emits parser-backed warning facts with stable source and fix spans for omitted header directions and for `style` targets that auto-create unknown nodes, and the same warning now reaches LSP `publishDiagnostics` with a stable rule id and range while render models still use effective `TB`; `merman-analysis` rule descriptors now carry `origin`, `default_profile`, and `evidence`, public config supports `core`/`recommended`/`strict` lint profiles plus explicit `enable_rules`, and authoring rules such as `merman.authoring.config.prefer_init_directive`, `merman.authoring.config.prefer_frontmatter_config`, and `merman.authoring.flowchart.explicit_direction` are recommended-profile hints rather than default core diagnostics. `prefer_frontmatter_config` now matches both `init` and `initialize` directive aliases and carries a preferred migration quickfix that rewrites init/initialize directive config into YAML frontmatter, while `prefer_init_directive` remains the smaller alias-reminder fix when the frontmatter rule is disabled. The migrated frontmatter quickfix now preserves existing frontmatter fields and reuses shared directive spans so analysis, CLI, LSP code actions, and future parser-backed editor features can share one directive locator. `merman.compatibility.config.deprecated_flowchart_html_labels` now carries a preferred quickfix that promotes `flowchart.htmlLabels` into root-level `htmlLabels`; `merman.compatibility.config.deprecated_external_diagram_loading` remains fixless because it is an API/config migration rather than a local rewrite. The governed lint rule catalog is now exported through Rust, CLI `lint-rules`, binding-core, C FFI, JNI, UniFFi, WASM, Web TS, Flutter/Dart, Apple Swift, and LSP custom request `merman/ruleCatalog`; LSP clients can also discover accepted analysis/lint settings through `merman/configSchema`. The `htmlLabels` compatibility rule remains scoped to upstream-backed `flowchart.htmlLabels` via the shared `source_directives` path matcher, giving future config lint, rewrite, completion, and hover work a reusable init-directive key-path locator instead of per-rule scanning. Parser-backed expected syntax now covers ER id lists, class diagram identifier positions, flowchart shapeData value positions, mindmap node id / payload spans, and Gantt task-id / dependency / click-target spans, so flowchart, mindmap, and Gantt completion can stay parser-backed even for multiline or nested label bodies instead of relying only on line-prefix heuristics. Mindmap node labels now emit explicit payload symbols, so label text no longer competes with node-id navigation or rename, and the mindmap fallback scan keeps labels out of completion ids. Markdown Mermaid fences now materialize as multiple snapshot fences, and completion stays fence-local across multi-fence Markdown documents instead of collapsing into a single fence or leaking ids between diagrams. Parser-backed expected syntax now also covers flowchart direction values, hyphenated node ids, and recovered `A-->`-style operator tails, so both `direction` keywords and dash-bearing node ids like `wi-fi` stay completion-visible instead of being treated as operator-only input. The mature roadmap's finish line is now explicit: capability matrix, public rule catalog/config schema, and fixture/performance gates all have to agree before the surface can be called mature.
- In progress: U2 now has parser-backed mindmap and Gantt entity-link slices in addition to the flowchart shape/value/direction/shape-trigger work. `mindmap` now keeps node ids and explicit labels as separate parser-backed regions, and Gantt task ids, dependency refs, and click targets now share a single rename/reference group. Flowchart now also emits a core warning fact for `style` directives that auto-create unknown nodes, and that same warning is now visible in LSP publishDiagnostics, which reinforces the family-local parser strategy: keep parser-generator choices local to the family instead of doing a blanket LALRPOP rewrite. U4 still needs broader lint-rule coverage and governance hardening, and U8 now has capability-matrix, Markdown multi-fence, and fence-local completion gates but still needs fixture/performance/docs gates before the roadmap can be called mature.
