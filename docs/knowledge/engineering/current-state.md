# Current State

- Goal: 推进 `docs/plans/2026-06-24-003-refactor-mature-mermaid-lsp-roadmap-plan.md`，把 merman 做成成熟的全功能 Mermaid LSP 与 lint/analysis 产品级基建；允许破坏性内部重构，持续移除启发式解析路径，完成 parser-backed semantic facts、诊断、补全、hover、symbols、definition/references、rename、code action、semantic tokens、配置/包装、测试与文档门槛。
- Audit snapshot: the shared LSP transport is already mature, and the remaining broad family-coverage gap for the current supported set is closed. Twenty-eight diagram families have parser-backed or source-mapped editor facts in the first-class capability matrix, including `Block`, `C4`, and `ZenUML`; `error` stays internal-only. Info, Pie, Packet, and XY Chart are intentionally payload-first, so navigation and rename surfaces stay sparse by design.
- Branch: feat/diagnostics-analysis-contract
- Last verified: 2026-06-28 (`cargo fmt --all --check`, `cargo nextest run -p merman-lsp --test capabilities`, `cargo nextest run -p merman-lsp --test document_store`, `cargo check -p merman-lsp --tests`, `cargo nextest run -p merman-core c4_editor_facts zenuml_editor_facts`)
- Done: `workspace/symbol` now reuses tracked outline projections from document snapshots; semantic tokens have full/range/delta handlers over parser-backed snapshot tokens; configuration changes trigger `workspace/semanticTokens/refresh` when supported; U3 navigation and rename route through entity-only typed reference groups; directive-oriented completion contexts no longer fall back to generic `flowchart TD` on directive lines; flowchart emits parser-backed warning facts with stable source and fix spans for omitted header directions and for `style` targets that auto-create unknown nodes, and the same warning now reaches LSP `publishDiagnostics` with a stable rule id and range while render models still use effective `TB`; `merman-analysis` rule descriptors now carry `origin`, `default_profile`, and `evidence`, public config supports `core`/`recommended`/`strict` lint profiles plus explicit `enable_rules`, and authoring rules such as `merman.authoring.config.prefer_init_directive`, `merman.authoring.config.prefer_frontmatter_config`, and `merman.authoring.flowchart.explicit_direction` are recommended-profile hints rather than default core diagnostics. `prefer_frontmatter_config` now matches both `init` and `initialize` directive aliases and carries a preferred migration quickfix that rewrites init/initialize directive config into YAML frontmatter, while `prefer_init_directive` remains the smaller alias-reminder fix when the frontmatter rule is disabled. The migrated frontmatter quickfix now preserves existing frontmatter fields and reuses shared directive spans so analysis, CLI, LSP code actions, and future parser-backed editor features can share one directive locator. `merman.compatibility.config.deprecated_flowchart_html_labels` now carries a preferred quickfix that promotes `flowchart.htmlLabels` into root-level `htmlLabels`; `merman.compatibility.config.deprecated_external_diagram_loading` remains fixless because it is an API/config migration rather than a local rewrite. The governed lint rule catalog is now exported through Rust, CLI `lint-rules`, binding-core, C FFI, JNI, UniFFi, WASM, Web TS, Flutter/Dart, Apple Swift, and LSP custom request `merman/ruleCatalog`; LSP clients can also discover accepted analysis/lint settings through `merman/configSchema`. The `htmlLabels` compatibility rule remains scoped to upstream-backed `flowchart.htmlLabels` via the shared `source_directives` path matcher, giving future config lint, rewrite, completion, and hover work a reusable init-directive key-path locator instead of per-rule scanning. Parser-backed expected syntax now covers ER id lists, class diagram identifier positions, flowchart shapeData value positions, mindmap node id / payload spans, state node identifiers / id-lists / payload spans, and Gantt task-id / dependency / click-target spans, so flowchart, mindmap, state, and Gantt completion can stay parser-backed even for multiline or nested label bodies instead of relying only on line-prefix heuristics. Mindmap node labels now emit explicit payload symbols, so label text no longer competes with node-id navigation or rename, and the mindmap fallback scan keeps labels out of completion ids. Markdown Mermaid fences now materialize as multiple snapshot fences, and completion stays fence-local across multi-fence Markdown documents instead of collapsing into a single fence or leaking ids between diagrams. Parser-backed expected syntax now also covers flowchart direction values, hyphenated node ids, and recovered `A-->`-style operator tails, so both `direction` keywords and dash-bearing node ids like `wi-fi` stay completion-visible instead of being treated as operator-only input. The mature roadmap's finish line is now explicit: capability matrix, public rule catalog/config schema, and fixture/performance gates all have to agree before the surface can be called mature. `classDef` class-diagram names are now outline-only instead of completion-visible entities, and class-diagram `class` / `cssClass` / `link` / `callback` / `:::` directive prefixes are recognized consistently in parser facts, analysis fallback, and LSP completion gating. Text-scan fallback now distinguishes classify-only directive lines (`class`, `classDef`, `style`, `linkStyle`, `click`, `section`) from payload-only sequence/gantt directives, which reduces false node-id leaks while preserving useful outline entries for class and gantt section lines. The crate README and diagnostic-protocol docs now match the wired LSP surface, and the capability matrix explicitly distinguishes the first-class maturity contract from the broader parser/render registry.
