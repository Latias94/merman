hover/documentSymbol/definition/references/prepareRename/rename now consume the shared `merman-analysis::FenceTextIndex`; LSP snapshot no longer owns its own completion/structure scan; flowchart fences now prefer parser-backed `merman-core::EditorSemanticFacts` with original-text byte spans for node ids, subgraph headers, and directive prefixes; sequence fences now emit parser-backed actor/participant/message-endpoint/note/box facts with complete/recovered provenance; state editor facts now come from a single token-backed `StateEditorEvent` stream for both complete and recovered inputs, with the LALRPOP parse result only selecting provenance; the state event stream emits state/reference/fork/join/choice entities, `class`/`style`/`click` state targets, `classDef` outline facts, and style, class reference, click URL/tooltip, accessibility text, display-label, description, relation-label, and note text payload-only spans; class fences now emit parser-backed class/namespace/relation/member-owner/directive-target/interaction-target facts plus class-body, inline member, and classDef outline spans, while annotation names, cssClass/style/classDef style values, inline class references, interaction URLs/tooltips/link targets, callback names/args, relation labels, note text, accessibility title/description values, class display labels, and relation multiplicity strings are preserved as payload spans, all from the class lexer token stream with complete/recovered provenance; ER fences now emit parser-backed entity/relationship endpoint/attribute/class/style/classDef facts with complete/recovered provenance, with attribute names projected as outline-only facts and attribute type/key/comment spans preserved as payload facts; incomplete attribute blocks no longer loop EOF recovery; mindmap now emits parser-backed node facts from a shared parser event stream, preserves class/icon decoration semantics, exposes class/icon decoration values as payload-only spans, and LSP records `ParserComplete`/`ParserRecovered` instead of falling back to `TextScan`; Gantt now emits parser-backed task id, section outline, directive payload, click payload, single-line and multiline accessibility payload, `after`/`until` dependency, `click` target, and directive-prefix facts with complete/recovered provenance, while keeping `section`, directive values, accessibility text, click URLs, callbacks, and callback args out of node-id completion; unclosed Gantt `accDescr { ...` blocks preserve recovered payload facts instead of silently dropping the text; `EditorSemanticFacts` now carries parser-backed recovery diagnostics, grammar-backed flowchart/sequence/state/class/ER editor parsers attach LALRPOP error messages plus token/EOF byte spans, and `merman-analysis` projects them as `merman.parse.recovered_editor_facts` warnings consumed by LSP/CLI/FFI/WASM through the existing diagnostics payload; FFI and WASM test mirrors were fixed to match the existing `analyze_json` surface; `merman-lsp` now opts into core full/host features so product LSP detection includes mindmap rather than the tiny registry; Python compatibility still resolves `python` to `python3` on macOS.
