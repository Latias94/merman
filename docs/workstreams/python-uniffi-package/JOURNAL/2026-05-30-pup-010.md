# PUP-010 - Scope And Evidence Freeze

Opened the `python-uniffi-package` workstream.

Key decisions:

- Treat source-only bindgen as insufficient for Python package readiness.
- Use a staged package layout with generated Python module plus cdylib in the same package
  directory.
- Make a real Python import/call smoke the main gate.
- Keep generated Python and native libraries out of git.
- Leave ASCII files untouched because they are owned by parallel user work.

Evidence:

- `python --version` reports Python 3.13.11.
- UniFFI 0.31.1 Python loader template places the dynamic library lookup relative to the generated
  module file.
