# F115-070 Flowchart ELK Gate Policy

Date: 2026-06-01

## Summary

The supported Flowchart 11.15 DOM matrix is green against the fresh Mermaid 11.15 SVG target. The
only remaining red item was `flowchart/upstream_html_demos_flowchart_elk_flowchart_elk_001`, which
requires ELK layout support that the local headless Flowchart layout path does not implement.

F115-070 closes this by documenting `flowchart-elk` as out of the current supported matrix and by
adding a narrow `compare-svg-xml` skip reason for that one upstream fixture. The skip is a local
support-policy decision, not an upstream regeneration failure.

## Evidence

- `cargo test -p xtask svg_xml_compare_skip_reason`: passed, 2 tests.
- `cargo run -p xtask -- compare-svg-xml --check --diagram flowchart --upstream-root target/upstream-svgs-11-15-flowchart --dom-mode parity --dom-decimals 3`:
  passed.
- `target/compare/xml/xml_report.md` reports `Mismatches (0)` and `Skipped (1)` for the documented
  `flowchart-elk` fixture.

## Decision

`flowchart-elk` should be handled by a dedicated ELK layout workstream if/when it enters the
supported headless rendering matrix. It should not block ordinary Flowchart DOM convergence or
stored baseline refresh.
