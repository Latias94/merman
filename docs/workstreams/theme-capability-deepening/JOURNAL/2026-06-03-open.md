# Theme Capability Deepening Lane Open

Date: 2026-06-03

## Why A New Lane Exists

`theme-parity` is complete and already documented its remaining work as a split follow-up:

- broader render-side resolver migration;
- stronger theme capability seams;
- chart/palette-oriented work that does not belong in raw Mermaid preset parity.

The repository architecture audit independently confirmed the same issue as `ARCH-013`: effective
config and theme access is still scattered.

## What This Lane Owns

- render-side presentation theme architecture in `merman-render`;
- deletion of duplicated renderer fallback logic where a deeper shared seam is justified;
- capability-oriented theme growth that belongs to rendering rather than host postprocessing.

## What This Lane Does Not Own

- host/editor palette rewriting;
- redoing `merman-core` theme preset compatibility;
- reopening Mermaid theme registry parity.
