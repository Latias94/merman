# ACEG-060 Public Gates And Closeout

## Summary

Closed the ASCII class/ER graph layout lane after verifying the public package, library, and CLI
entry points still expose the shipped layered relationship layouts.

The public docs now describe classDiagram and erDiagram text output as bounded terminal-native
subsets with boxes, labels, single relationships, and layered chain/star multi-relationship layouts.
Unsupported dense, crossing, cyclic, parallel, spanning-level, and unrelated graph shapes remain
explicit diagnostics so relationships are not silently omitted.

## Review

No blocking closeout findings remain. The main residual architecture concern is duplication between
the class and ER layered placement planners. Keeping that code inside the adapters is acceptable for
this lane because relationship semantics stay typed-model-owned and the shared `relation_graph`
module remains terminal-layout-only.

Split follow-ons:

- Extract a shared layered planner only if the next class/ER topology slice needs the same level
  assignment, ordering, and crossing detection.
- Add dense/crossing class/ER topology routing separately, with tests before broadening support.
