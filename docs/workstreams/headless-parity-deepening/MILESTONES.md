# Headless Parity Deepening - Milestones

## M0 - Lane Is Trustworthy

- Workstream docs agree on scope, non-goals, and first slice.
- Architecture issue audit and active residual lane are referenced explicitly.

## M1 - Baseline Truth Is Local

- Active Mermaid baseline facts no longer depend on remembering a mix of docs, generated names,
  and registry constructor labels.

## M2 - Residuals Are Classified

- Top active parity-root residual buckets are classified by kind and seam.
- The lane can distinguish “fix next” from “bounded headless residual” without fake percentage
  claims.

## M3 - Measurement / Root Bounds Seams Exist

- New parity fixes in Sequence / Architecture / Flowchart / Class can land through explicit
  measurement/root-bounds seams.
- New ad hoc text constants are rejected unless they are seam-owned and justified.

## M4 - Layout Engine Audit Produces Decisions

- `manatee` / `dugong` audit slices produce source-backed conclusions:
  source gap, input seam mismatch, or bounded residual.

## M5 - Future Scope Is Intentional

- Unsupported Mermaid family admission has a rubric and priority order.
- Follow-on family work can open cleanly without re-litigating headless fit.

## M7 - Visible Rendering Is Trustworthy

- Structural DOM parity no longer hides unreadable output.
- Missing diagram-specific theme CSS, invisible labels, blank renders, and lost semantic colors are
  fixed before small root viewport residuals.

## M8 - Baseline Preparation Is Closed

- Current-facing baseline version surfaces route through the pinned Mermaid 11.15 baseline truth.
- Stored upstream SVG drift was classified and refreshed only for stale family or fixture sets.
- Full implemented-matrix DOM parity, layout snapshots, and boundary renderability are green before
  returning to HPD-080 or HPD-050 work.
