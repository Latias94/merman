# 2026-06-01 - Open Theme Parity Refactor

Opened the lane after comparing Mermaid's theme initialization and theme preset source with
Merman's current core, WASM, and playground paths.

Initial execution starts with TPR-020 because `theme: default` is the broadest correctness gap and
the safest foundation for later fallback deletion.

TPR-020 completed in the same session. Default theme expansion now populates core Mermaid theme
variables, keeps user overrides, and makes unknown themes degrade through the default theme path.
The class/block scoped-color tests were updated to expect Mermaid's default class/node text color
`#131300` instead of the old fallback `#333`.

TPR-030 completed by extracting shared theme helper code for themeVariables map cloning, Mermaid's
default font-family value, and `mkBorder` HSL derivation. Core theme tests still pass without
changing existing base/dark/forest/neutral assertions.

TPR-040 completed by adding `SvgTheme` and migrating Class, Block, and Flowchart CSS callers to the
shared resolver. Renderer SVG regression tests for block/class/flowchart passed.

TPR-050 completed by supporting Mermaid `themeCSS` as scoped diagram-owned CSS after parity SVG
rendering. The scoped CSS postprocessor now handles Mermaid-style `&` selectors, nested grouping
at-rules, unsupported top-level at-rule removal, keyframes, and Mermaid hash placeholders used by
CSS color values.

TPR-060 completed by single-sourcing supported themes through core, bindings, WASM, `@merman/web`,
and playground consumers. The supported list is now `default`, `base`, `dark`, `forest`, and
`neutral`; unknown themes normalize to `default`.

Closeout completed with targeted Rust, WASM, and frontend gates. Broad theme fixture coverage,
remaining diagram-specific resolver migrations, and `neo/redux` theme families were split to
follow-up parity work instead of being hidden inside this lane.
