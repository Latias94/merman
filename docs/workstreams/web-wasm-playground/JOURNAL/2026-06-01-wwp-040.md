# WWP-040 - Playground Integration

Date: 2026-06-01
Status: DONE

## Summary

Integrated the reference live editor as `playground` and wired it to the first-class `@merman/web`
package. The normal preview path now loads generated WASM and renders SVG through Rust rather than
the old mock loader.

## Implementation Notes

- Copied the static Vite/React editor from `repo-ref/merman-page`, excluding obsolete placeholder
  Rust and generated lock artifacts.
- Added `@merman/web` as a local file dependency and changed the loader to initialize WASM once.
- Added `renderAscii` support through `merman-bindings-core`, `merman-wasm`, and `platforms/web`.
- Replaced runtime `std::time` usage in the browser render chain with `web-time` after smoke tests
  exposed `Instant::now()` panicking in wasm.
- Directed TypeScript build info into `node_modules/.tmp` and ignored playground generated output.

## Evidence

```bash
cargo check -p merman-wasm --target wasm32-unknown-unknown
npm run build --prefix platforms/web
npm run prepack --prefix platforms/web
npm run build --prefix playground
cargo nextest run -p merman-wasm -p merman-bindings-core
cargo nextest run -p dugong -p manatee
cargo fmt --check
git diff --check
```

Browser smoke passed via local preview at `http://127.0.0.1:4173/merman/`: generated WASM was
requested and `.preview-container svg` rendered with 95 SVG child nodes. Screenshot:
`target/playground-preview/smoke.png`.

## Residual Risk

Full package suites have unrelated existing failures in `merman-render` math width and
`merman-core` snapshot parity. They were recorded in `EVIDENCE_AND_GATES.md` and should be triaged
outside WWP-040.
