# WWP-020 - Formal WASM Crate

Date: 2026-06-01
Status: DONE

## Summary

Added `crates/merman-wasm` as a first-class workspace crate. The crate is a thin `wasm-bindgen`
transport over `merman-bindings-core`, exposing SVG, semantic JSON, layout JSON, validation,
version helpers, and static capability lists.

## Implementation Notes

- `Cargo.toml` now includes `crates/merman-wasm` as a workspace member and dependency.
- `uuid` now enables the `js` feature for wasm-compatible v4 IDs.
- `crates/roughr` adds a wasm-target `getrandom` dependency with the `js` feature so its `rand`
  fallback compiles on `wasm32-unknown-unknown`.
- The WASM crate avoids renderer duplication; all behavior goes through `merman-bindings-core`.

## Evidence

```bash
cargo check -p merman-wasm
cargo check -p merman-wasm --target wasm32-unknown-unknown
cargo install wasm-bindgen-cli --version 0.2.108 --locked
wasm-pack build crates/merman-wasm --target web --out-dir ../../target/merman-wasm-pkg
cargo nextest run -p merman-wasm
cargo fmt --check
```

All gates passed. `wasm-pack` emitted a non-blocking license-file warning. Its first run failed
while auto-installing `wasm-bindgen-cli` without `--locked`; the locked manual install fixed that
tooling issue.
