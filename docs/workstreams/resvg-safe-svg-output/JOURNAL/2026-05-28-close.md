# 2026-05-28 Closeout

Closed the SVG output pipeline lane.

Implemented:

- `SvgPipeline::{parity, readable, resvg_safe}`.
- Public `SvgPostprocessor` with Cow-based input/output and deterministic ordering.
- Shared readable/raster/CLI raster routing through the pipeline.
- Resvg-safe cleanup for common downstream hazards observed from the Zed integration signal.
- User-facing docs and changelog entries.

Default SVG output remains parity-first. This is intentional: consumer cleanup is opt-in so
upstream Mermaid DOM comparisons stay meaningful.

Residual risks:

- The first resvg-safe implementation is string-oriented. This keeps the public API small, but a
  future internal XML/CSS parser may be worthwhile if profiling shows postprocessing cost matters.
- Theme/accent semantics remain host-specific and should not be generalized without a second
  non-Zed integration signal.
