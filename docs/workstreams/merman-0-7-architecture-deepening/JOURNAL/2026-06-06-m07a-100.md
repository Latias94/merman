# M07A-100 — Flowchart Single Semantic Source

Date: 2026-06-06

## Summary

Collapsed Flowchart semantic JSON parsing and typed render-model parsing around one internal
semantic source.

- Added `FlowchartSemanticSource` as the single owner of Flowchart parse/build/subgraph/semantic
  application output.
- `parse_flowchart` now projects that source into compatibility JSON.
- `parse_flowchart_model_for_render` now projects the same source into `FlowchartV2Model`.
- Preserved FlowDB ordering evidence such as `vertexCalls`.
- Kept the LALRPOP/lexer strategy and public compatibility JSON contract unchanged.
- Added a public API regression proving JSON and typed render paths share the same semantic facts
  for accessibility fields, class defs, tooltips, edge defaults, vertex calls, nodes, edges, and
  subgraphs.
