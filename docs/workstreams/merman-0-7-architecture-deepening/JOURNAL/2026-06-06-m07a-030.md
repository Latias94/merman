# M07A-030 — Adapter Adoption Of Render Operation

Status: Done
Date: 2026-06-06

## Summary

CLI Mermaid input rendering now calls `render_svg_with_pipeline_sync` with the CLI-owned
postprocess pipeline. This keeps postprocess ordering and metadata under the canonical render
operation for Mermaid input.

Existing SVG input still calls CLI-owned `postprocess_svg` before rasterization. That path is a
format Adapter concern because no Mermaid parse/layout/render operation is involved.

Bindings-core and raster already use operation-backed facade helpers after M07A-020, so no extra
code movement was needed there.

## Evidence

Passed:

- `cargo nextest run -p merman-bindings-core render_svg`
- `cargo nextest run -p merman-cli render`
- `cargo nextest run -p merman --features raster raster`
- `cargo fmt --all --check`

## Handoff

M07A-040 is ready. Audit public render convenience methods before deleting anything. Prefer deleting
or demoting only shallow pre-1.0 methods that do not make callers manually reconstruct the render
operation.
