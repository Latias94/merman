# M07A-077 Journal

Date: 2026-06-06
Status: Done

## Summary

- Added `--fixed-today <YYYY-MM-DD>` and `--fixed-local-offset-minutes <minutes>` to CLI parse,
  detect, layout, render, and top-level export flows.
- CLI `engine_for` now applies those values through the existing `Engine::with_fixed_today` and
  `Engine::with_fixed_local_offset_minutes` builder methods.
- Fixed the typed render-model parse path to use the same fixed local-time context as semantic JSON
  parsing. This matters for Gantt SVG output because rendering uses the typed render model path.
- Documented the flags as Rust-only deterministic options, not official `mmdc` compatibility
  options.

## Remaining Risks

- Bindings and other host adapters can already use the core `Engine` builder methods, but they do
  not yet expose equivalent public fixed-time options.
- This slice intentionally does not change Gantt date parsing, layout, theme, or SVG parity
  semantics beyond applying the existing fixed-time context consistently.
