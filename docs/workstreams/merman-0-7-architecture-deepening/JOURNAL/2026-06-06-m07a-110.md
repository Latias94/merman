# M07A-110 — Fence JSON Semantic Fallback As Adapter

Date: 2026-06-06

## Summary

Fenced `RenderSemanticModel::Json` as a compatibility adapter on the render-model path.

- Built-in non-error Mermaid families now fail fast if their typed render parser is missing.
- The built-in `error` diagram and custom diagram adapters still use JSON render fallback.
- Registry coverage now proves every pinned non-error semantic parser has typed render parser
  backing.
- Public compatibility JSON parsing/output was not changed.
- `check-alignment` no longer requires ignored local `fixtures/_deferred` investigation
  directories, while preserving `NormalizedWithDeferred` inventory metadata.

## Validation

- GREEN: `cargo nextest run -p merman-core`
- GREEN: `cargo nextest run -p merman-render`
- GREEN: `cargo fmt --all --check`
- GREEN: `cargo run -p xtask -- check-alignment`
- GREEN: `git diff --check`

## Notes

No ADR was needed because this does not change the public compatibility JSON contract. The change
only prevents built-in render-model parsing from silently dropping to JSON when a typed parser is
missing. Custom diagram adapters remain the extension point for JSON render fallback.

M07A-C4 is now complete. The next same-lane task is M07A-120 final verification and closeout.
