# M07A-076 Journal

Date: 2026-06-06
Status: Done

## Summary

- Raw `<svg>` input for PNG/JPG/PDF now uses a dedicated raster boundary instead of the
  Mermaid-generated SVG parity pipeline.
- The raw SVG path starts from `SvgPipeline::resvg_safe()` before CLI background/CSS
  postprocessors. The downstream conversion path still applies normal resvg-safe cleanup and size
  limits.
- CLI docs now state that raw SVG input is trusted input. This narrows the boundary without
  claiming a complete sanitizer for arbitrary uploaded SVG.

## Validation

- `cargo nextest run -p merman-cli raw_svg`
- `cargo nextest run -p merman-cli png`
- `cargo nextest run -p merman-cli pdf`
- `cargo run -p xtask -- check-alignment`
- `cargo fmt --all --check`

## Remaining Risks

- This does not replace host-side trust decisions for arbitrary uploaded SVG.
- `foreignObject` fallback transform parsing remains string-based and should be handled separately.
