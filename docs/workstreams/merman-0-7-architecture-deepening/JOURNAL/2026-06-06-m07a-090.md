# M07A-090 — Family-Owned Typed Semantic Sanitization

Date: 2026-06-06

## Summary

Moved typed render-model common DB sanitization field knowledge out of `Engine`.

- `RenderSemanticModel` now owns the typed sanitization dispatch.
- Family-owned typed models sanitize their own top-level `title`, `accTitle`, and `accDescr`
  fields where those fields exist.
- JSON fallback models still use `common_db::apply_common_db_sanitization`.
- Flowchart typed `accTitle` / `accDescr` and Treemap typed `title` now match the semantic JSON
  common DB sanitization boundary.

## Validation

- GREEN: `cargo nextest run -p merman-core sanitize`
- GREEN: `cargo nextest run -p merman-core`
- GREEN: `cargo fmt --all --check`
- GREEN: `git diff --check`

## Notes

This is a typed semantic ownership slice. It intentionally does not collapse Flowchart JSON and
typed projections; that remains M07A-100 because FlowDB ordering traces and SVG parity require a
separate focused pass.
