# merman Flutter/Dart FFI

Experimental Dart FFI wrapper for the `merman-ffi` C ABI.

This package loads the native library, checks the ABI version and struct sizes, then exposes a small
Dart API for SVG, semantic JSON, and layout JSON.

On Android this package also declares a Flutter plugin shim so Flutter apps can package native
`libmerman_ffi.so` slices generated by `platforms/android/build-android.py`.

## Local Smoke

Build the native library:

```bash
cargo build -p merman-ffi
```

Run the Dart smoke with an explicit native library path:

```bash
cd platforms/flutter
dart pub get
dart run example/smoke.dart ../../target/debug/libmerman_ffi.dylib
```

Use `../../target/debug/libmerman_ffi.so` on Linux and `../../target/debug/merman_ffi.dll` on
Windows.

The smoke example lives in [`example/smoke.dart`](example/smoke.dart).

## API

```dart
import 'package:merman/merman.dart';

final merman = Merman.open('path/to/libmerman_ffi.dylib');
final svg = merman.renderSvg('flowchart TD\nA[Hello] --> B[World]');
final semantic = merman.parseJson('flowchart TD\nA[Hello] --> B[World]');
final layout = merman.layoutJson('flowchart TD\nA[Hello] --> B[World]');
```

Packaging native libraries into Android, iOS, macOS, Windows, and Linux app/plugin bundles remains
follow-on work beyond the current Android plugin scaffold.

## Android Plugin Packaging

Build Android native slices first:

```bash
python3 platforms/android/build-android.py --targets aarch64-linux-android x86_64-linux-android
```

The Flutter Android plugin reads those generated libraries from:

```text
platforms/android/src/main/jniLibs
```

`jniLibs` is generated output and is not committed.

To verify plugin packaging through a temporary Android app:

```bash
python3 platforms/flutter/tool/android-smoke.py
```
